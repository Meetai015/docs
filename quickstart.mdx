## Get started in three steps

Get Gistly Call Analytics running locally and start analyzing your meetings in minutes.

### Step 1: Set up your local environment

<AccordionGroup>
  <Accordion icon="copy" title="Clone the repository">
    Clone the Gistly Call Analytics repository to get started:

    ```bash
    git clone https://github.com/your-org/gistly-call-analytics.git
    cd gistly-call-analytics
    ```

    The repository contains two main components:
    - **Frontend**: `meeting-notes-dashboard/` (Next.js/React)
    - **Backend**: `backend-meeting-notes/` (Python/Django)
  </Accordion>

  <Accordion icon="database" title="Set up the backend">
    1. Navigate to the backend directory:
       ```bash
       cd backend-meeting-notes
       ```

    2. Create a virtual environment:
       ```bash
       python -m venv venv
       source venv/bin/activate  # On Windows: venv\Scripts\activate
       ```

    3. Install dependencies:
       ```bash
       pip install -r requirements.txt
       ```

    4. Set up environment variables:
       ```bash
       cp .env.example .env
       # Edit .env with your configuration
       ```

    5. Run migrations:
       ```bash
       python manage.py migrate
       ```

    6. Start the backend server:
       ```bash
       python manage.py runserver
       ```

    The backend will be available at `http://localhost:8000`
  </Accordion>

  <Accordion icon="layout" title="Set up the frontend">
    1. Navigate to the frontend directory:
       ```bash
       cd meeting-notes-dashboard
       ```

    2. Install dependencies:
       ```bash
       npm install
       ```

    3. Set up environment variables:
       ```bash
       cp .env.local.example .env.local
       # Edit .env.local with your configuration
       ```

    4. Start the development server:
       ```bash
       npm run dev
       ```

    The frontend will be available at `http://localhost:3000`

    <Tip>The development server supports hot reload, so changes appear automatically.</Tip>
  </Accordion>
</AccordionGroup>

### Step 2: Configure your first integration

<AccordionGroup>
  <Accordion icon="video" title="Connect a meeting platform">
    Gistly integrates with popular meeting platforms. Choose one to get started:

    **Zoom**
    1. Create a Zoom app in the [Zoom Marketplace](https://marketplace.zoom.us/)
    2. Get your API key and secret
    3. Add credentials to your `.env` file:
       ```
       ZOOM_API_KEY=your_api_key
       ZOOM_API_SECRET=your_api_secret
       ```

    **Google Meet**
    1. Enable Google Meet API in [Google Cloud Console](https://console.cloud.google.com/)
    2. Create OAuth credentials
    3. Add credentials to your `.env` file

    **Microsoft Teams**
    1. Register an app in [Azure Portal](https://portal.azure.com/)
    2. Configure Graph API permissions
    3. Add credentials to your `.env` file
  </Accordion>

  <Accordion icon="database" title="Set up the database">
    Gistly uses PostgreSQL for data storage. Choose your setup:

    **Local Development (Docker)**
    ```bash
    docker run --name gistly-postgres \
      -e POSTGRES_PASSWORD=gistly123 \
      -e POSTGRES_DB=gistly \
      -p 5432:5432 \
      -d postgres:15
    ```

    **Production**
    - Set up a managed PostgreSQL instance (AWS RDS, Google Cloud SQL, etc.)
    - Update `DATABASE_URL` in your `.env` file

    <Tip>For development, Docker is the quickest way to get started.</Tip>
  </Accordion>
</AccordionGroup>

### Step 3: Run your first analysis

<Accordion icon="rocket" title="Analyze your first meeting">
  Now you're ready to analyze your first meeting:

  1. **Import a meeting**
     ```bash
     # Via the API
     curl -X POST http://localhost:8000/api/meetings/import \
       -H "Authorization: Bearer YOUR_API_KEY" \
       -d '{"platform": "zoom", "meeting_id": "YOUR_MEETING_ID"}'
     ```

  2. **View the dashboard**
     - Open `http://localhost:3000`
     - Navigate to the Analytics dashboard
     - See insights from your imported meeting

  3. **Explore features**
     - View meeting transcripts
     - Check action items
     - Analyze participant engagement
     - Review sentiment trends

## Troubleshooting

<AccordionGroup>
  <Accordion icon="triangle-exclamation" title="Backend won't start">
    Common issues and solutions:

    1. **Port already in use**
       ```bash
       # Change the port
       python manage.py runserver 8001
       ```

    2. **Database connection error**
       ```bash
       # Check PostgreSQL is running
       docker ps | grep postgres

       # Verify DATABASE_URL in .env
       ```

    3. **Migration errors**
       ```bash
       # Reset migrations
       python manage.py migrate --fake-initial
       ```
  </Accordion>

  <Accordion icon="triangle-exclamation" title="Frontend won't start">
    Common issues and solutions:

    1. **Port already in use**
       ```bash
       # Kill the process on port 3000
       lsof -ti:3000 | xargs kill -9
       ```

    2. **Dependency errors**
       ```bash
       # Clear cache and reinstall
       rm -rf node_modules package-lock.json
       npm install
       ```

    3. **API connection errors**
       ```bash
       # Verify backend is running
       curl http://localhost:8000/api/health

       # Check NEXT_PUBLIC_API_URL in .env.local
       ```
  </Accordion>
</AccordionGroup>

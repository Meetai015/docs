## Overview

Gistly embraces AI-assisted development to accelerate productivity and maintain code quality. This guide covers the AI tools and workflows used in the project.

## Supported AI Tools

### Cursor IDE

[Cursor](https://cursor.sh) is an AI-first code editor built on VS Code with native AI integration.

#### Setup

1. **Install Cursor**
   ```bash
   # Download from https://cursor.sh
   # Or use Homebrew
   brew install --cask cursor
   ```

2. **Configure Cursor for Gistly**

   Open Cursor Settings (Cmd+,) and configure:

   ```json
   {
     "cursor.ai.enabled": true,
     "cursor.ai.model": "claude-3.5-sonnet",
     "cursor.ai.temperature": 0.3,
     "cursor.ai.maxTokens": 2000
   }
   ```

3. **Open Gistly in Cursor**
   ```bash
   cursor /Users/shishirmehto/git/gistly-call-analytics
   ```

#### Features

- **AI Chat** (Cmd+L): Ask questions about code, get explanations
- **Code Generation** (Cmd+K): Generate code from natural language
- **Refactoring** (Cmd+I): Intelligent code refactoring
- **Multi-file Editing**: Edit multiple files simultaneously
- **Context Awareness**: AI understands your entire codebase

#### Workflow

1. **Code Review**
   ```
   Use Cursor to review code changes:
   - Select the code
   - Press Cmd+L
   - Ask: "Review this code for bugs and improvements"
   ```

2. **Generate Tests**
   ```
   Select a function
   Press Cmd+K
   Prompt: "Write unit tests for this function"
   ```

3. **Debug Issues**
   ```
   Use AI Chat to debug:
   - Describe the issue
   - Paste error messages
   - Get suggested fixes
   ```

### Claude Code

[Claude Code](https://claude.ai/code) is Anthropic's official CLI for Claude, designed for software development tasks.

#### Setup

1. **Install Claude Code CLI**
   ```bash
   npm install -g @anthropic/claude-code
   ```

2. **Authenticate**
   ```bash
   claude-code auth
   ```

3. **Navigate to Gistly**
   ```bash
   cd /Users/shishirmehto/git/gistly-call-analytics
   claude-code
   ```

#### Features

- **Context-Aware**: Understands your entire codebase
- **Multi-File Operations**: Edit multiple files at once
- **Git Integration**: Smart commit messages and PR reviews
- **Code Analysis**: Deep code understanding and refactoring
- **Testing**: Automated test generation and debugging

   Claude will:
   - Analyze the code
   - Identify the issue
   - Implement a fix
   - Add tests
   ```

3. **Documentation**
   ```
   Prompt: "Add docstrings to all functions in analytics.py"
   Claude will:
   - Add comprehensive docstrings

### Windsurf

[Windsurf](https://windsurf.ai) is an AI-powered IDE extension for VS Code.

#### Setup

1. **Install VS Code Extension**
   ```
   Search "Windsurf AI" in VS Code extensions
   Click "Install"
   ```

2. **Configure Windsurf**

   Create `.windsurfrc.json` in project root:

   ```json
   {
     "model": "claude-3.5-sonnet",
     "temperature": 0.3,
     "maxTokens": 2000,
     "contextWindow": 200000,
     "projectContext": {
       "include": [
         "meeting-notes-dashboard/**/*",
         "backend-meeting-notes/**/*"
       ]
     }
   }
   ```

3. **Restart VS Code**

#### Features

- **Real-time Suggestions**: AI code completions as you type
- **Code Explanation**: Understand complex code instantly
- **Refactoring**: Intelligent code improvements
- **Documentation**: Auto-generate docs from code
- **Testing**: Generate test cases automatically

#### Workflow

1. **Real-time Assistance**
   ```
   As you type, Windsurf provides:
   - Code completions
   - Function suggestions
   - Parameter hints
   - Error fixes
   ```

2. **Code Analysis**
   ```
   Right-click on code
   Select "Explain with Windsurf"
   Get detailed explanations
   ```

3. **Refactoring**
   ```
   Select code
   Right-click
   Choose "Refactor with Windsurf"
   Apply suggested improvements
   ```

## Best Practices

### Prompt Engineering

1. **Be Specific**
   ```
   Good: "Create a React component for meeting insights with charts for duration, participants, and action items"
   Bad: "Create a meeting component"
   ```

2. **Provide Context**
   ```
   Good: "Add error handling to the login function. Use the existing error handling pattern in auth.py. Handle network errors, invalid credentials, and server errors."
   Bad: "Add error handling"
   ```

3. **Request Tests**
   ```
   Always ask for tests when generating code:
   "Create a function to calculate meeting scores and include unit tests for edge cases"
   ```

### Code Quality

1. **Review AI Output**
   - Always review generated code
   - Run tests before committing
   - Check for security issues
   - Ensure code style consistency

2. **Iterate**
   - Use AI for initial drafts
   - Refine with human expertise
   - Combine multiple AI suggestions
   - Learn from AI patterns

3. **Test Thoroughly**
   - Run all tests
   - Check edge cases
   - Verify performance
   - Test manually

### Security

1. **Never Share Secrets**
   - Don't paste API keys in AI prompts
   - Use environment variables
   - Sanitize sensitive data
   - Use placeholder values

2. **Review Generated Code**
   - Check for vulnerabilities
   - Validate input handling
   - Review authentication logic
   - Verify data sanitization
